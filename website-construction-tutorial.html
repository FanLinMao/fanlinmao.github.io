GitHub Pages和Hexo博客搭建详细教程

发表于 2020-07-08 | 分类于 博客教程 | | 浏览 次
字数统计: 5.1k | 阅读时长 ≈ 27 分钟

拥有github账号

访问github网址，点击sign up注册账号，如果有账户忽略此步骤。

新建git仓库

登录github，然后新建仓库，仓库名建议为username.github.io。为什么建议这个，假如你新建仓库名为blog，那么在后面生成的Github Pages的首页访问链接为https://username.github.io/blog。
如果新建的仓库名为前者username.github.io，生成的Github Pages的首页访问链接则为https://username.github.io，访问的URL就是GitHub Pages的域名，显得不会多余。

注意：这里的username指的是你的账户名。

设置仓库为GitHub Pages

进入上一步新建的仓库中，点击settings，看到github pages选项栏。
如果在步骤2)中，没有选择初始化一个README.md文件或者新建空的仓库(里面没有任何文件)，就会设置不了Github Pages，也会提示如下所示的信息。

安装Node.js

下载Node.js安装包。

注意：安装时，请确保已选中“添加到PATH”（默认情况下已选中）

安装git

安装git是方便操作git命令以及hexo的一些命令，方便快速的发布到git仓库中。
根据系统类型，下载对应的git安装包：https://git-scm.com/downloads

安装Hexo

Hexo是一个快速，简单且功能强大的，基于Node.js的博客框架。使用Markdown（或其他标记语言）编写页面，然后Hexo会在几秒钟内生成带有精美主题的静态文件。

然后打开git bash工具，操作如下命令安装hexo：

1.全局安装hexo

$ npm install hexo-cli -g

2.切换到你想要的目录

$ cd ~/Desktop

3.初始化一个hexo博客，文件名自定义后会自动创建，这里假设为hexo

$ hexo init hexo
$ cd hexo

初始化完后，文件目录如下(除public外)：

4.安装依赖

$ npm install

5.本地运行

$ hexo server

执行完命令后，直接输入localhost:4000，可以看到默认的hexo博客主题页面。

hexo常用命令：
$ hexo clean #清除缓存文件db.json和生成的public目录
$ hexo generate 或 hexo g #生成静态页面至public目录
$ hexo server 或 hexo s #开启预览访问端口（默认端口4000，’ctrl + c’关闭server）
$ hexo deploy 或 hexo d #部署到GitHub
$ hexo help #查看帮助
$ hexo version #查看Hexo的版本
$ hexo new “postName” #新建文章
$ hexo new page “pageName” #新建页面

然后打开_config.yml配置文件，这个配置文件是在项目根目录下，称为站点配置文件，这个文件主要是设置网站的信息，比如标题、作者等信息。
后续还有themes中的主题文件中包含_config.yml配置文件，这个配置文件是在主题的根目录下（如themes/yilia/_config.yml），称为主题配置文件，这个文件主要配置网站页面布局等信息。

选择喜欢的主题

hexo提供了很多主题，具体的可查看和预览官网 主题。
介绍几款个人比较喜欢的：

  • hexo-theme-jekyll
  • hexo-theme-yilia
  • hexo-theme-next

1.在创建的博客项目主目录下

$ git clone https://github.com/litten/hexo-theme-yilia.git themes/yilia

也可手动下载yilia主题，下载完后，直接将文件放在项目目录下的themes下面。

2.修改站点配置文件：将theme: landscape改为theme: yilia，然后重新先后执行执行hexo clean和hexo g来重新生成，最后hexo s本地部署后再访问localhost:4000。

主题的切换：当你想换哪个主题可以直接切换哪个主题即可，前提是你都得下载好主题文件放在项目根目录下的themes文件中。

配置SSH key

配置这个是为了可以直接提交代码到github仓库中。

先全局配置下你的git用户名和邮箱：

$ git config --global user.name "xxx"// 你的github用户名，非昵称
$ git config --global user.email  "xxx@qq.com"// 填写你的github注册邮箱

查看本地ssh key：

$ ls ~/.ssh #检查本机已存在的ssh密钥

如果.ssh这个目录下有id_rsa.pub文件，直接将文件中的密钥复制到github的SSH keys中，没有则需要生成密钥：

$ ssh-keygen -t rsa -C "github注册邮箱地址"

生成后，再复制到SSH keys中。

部署博客

在项目根目录下执行：

$ hexo clean  #清理一下
$ hexo g  #生成public文件，public里都是静态文件
$ hexo s  #此步可省

部署之前需要安装部署插件：

$ npm install hexo-deployer-git --save

再确认站点配置文件是否配置git地址：

deploy:
  type: git
  repository: git@github.com:FanLinMao/fanlinmao.github.io.git
  branch: master

执行部署：

$ hexo d  #部署

绑定域名

到这里，我们可以使用username.github.io来访问了，但是网站想要个性点，可以自己买个域名。
买了域名后，需要进行实名认证，如果没有实名，域名解析状态会显示不正常，并且访问也会404错误的。
域名配置最常见有2种方式，CNAME和A记录，CNAME填写域名，A记录填写IP，由于不带www方式只能采用A记录，所以必须先ping一下你的用户名.github.io的IP，然后到你的域名DNS设置页，将A记录指向你ping出来的IP，将CNAME指向你的用户名.github.io，这样可以保证无论是否添加www都可以访问，笔者买的阿里云的.top域名，首年10元。设置如下：

保留一个CNAME，CNAME中写你的域名，将 CNAME文件放到source目录下

绑定完后，再重新部署下，hexo clean,hexo g,hexo d你会看到仓库中github pages的部署地址已改为你的域名，可以通过自己买的域名访问了。
